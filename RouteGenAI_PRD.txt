1. EXECUTIVE SUMMARY

RouteGen AI is an intelligent model routing middleware that intercepts every prompt in an AI pipeline and automatically dispatches it to the most cost-effective LLM tier without sacrificing output quality. Built by Team Neural Nexus for HACKINDIA Track 03, RouteGen AI addresses one of the most pressing inefficiencies in modern AI deployments: the blanket use of expensive frontier models for tasks that smaller, faster models can handle equally well.

By classifying task complexity in real-time and routing across three model tiers (Small, Large, and Reasoning), RouteGen AI achieves up to 80% reduction in API costs while maintaining quality parity through an automatic fallback mechanism and full observability via Langfuse.

------------------------------------------------------------------------

2. PROBLEM STATEMENT

2.1 Core Problem

Current AI pipelines route every single task, from complex multi-step reasoning down to basic JSON formatting, through massive expensive frontier models like GPT-4o or Claude Opus. This one-size-fits-all approach creates three compounding problems.

Cost Explosion: API bills skyrocket as scale increases. Simple tasks like data extraction or reformatting cost the same per token as deep reasoning tasks, meaning companies pay frontier model prices for work a small model could handle.

Latency Overhead: Trillion-parameter models introduce unnecessary latency for basic tasks that a 7B parameter model can resolve in milliseconds. Every user request waits longer than it needs to.

Zero Observability: There is no visibility into which model handled which task, why it was selected, or how much it cost, making debugging and cost optimization nearly impossible.

2.2 Industry Gap

The industry currently lacks a production-ready intelligent routing layer that can evaluate task complexity in real-time at each orchestration node, automatically select the cheapest model capable of meeting quality requirements, log all routing decisions with full cost and performance traceability, and fall back gracefully when a smaller model fails to meet quality thresholds.

3.1 Primary Goals

Quality Retention target is greater than 90 percent versus the baseline as measured by BERTScore and DSPy assertions.

Router Latency Overhead must be less than 10 milliseconds per routing decision as measured by Langfuse trace latency.

Fallback Accuracy must be 100 percent escalation on any assertion failure, verified by automated test suite.

6. SYSTEM ARCHITECTURE

6.1 Architecture Overview

RouteGen AI operates as a middleware intercept layer sitting between the orchestration framework (LangGraph) and all LLM providers. Every node in the pipeline passes through RouteGen AI before any model call is made.

The pipeline flow works as follows. A JSON prompt or user input enters the system through the Input Gateway and API Endpoint. It is first checked against the Semantic Cache in a vector database; if a similar query exists with a cosine similarity above 0.95, the cached result is returned immediately at zero cost. If there is no cache hit, the Compositional Analysis and Complexity Grading Engine scores the prompt on a scale of 1 to 10. The Dynamic Routing and Dispatch module then sends the prompt to the appropriate model tier. The Small Tier (score 1 to 4) uses models like Llama 3, Claude Haiku, and Gemini Flash. The Large Tier (score 5 to 7) uses models like GPT-4o and Claude Sonnet. The Reasoning Tier (score 8 to 10) uses models like o3-mini and Claude Opus. If the output fails a DSPy quality assertion, the Auto-Fallback Self-Healing Loop automatically escalates to the next tier. The final output is assembled and delivered to the user. Throughout the entire process, every event is sent to the Observability and Telemetry Hub powered by Langfuse, which feeds the Real-Time Dashboard.

6.2 Complexity Scoring

Every prompt is scored on five weighted dimensions before routing. Token Length accounts for 20 percent and measures the estimated input and output token count. Ambiguity accounts for 20 percent and measures the presence of underspecified or open-ended instructions. Domain Depth accounts for 25 percent and measures the density of specialized vocabulary in technical, legal, or scientific domains. Output Format accounts for 15 percent and distinguishes structured outputs like JSON or code from free-form text. Reasoning Required accounts for 20 percent and identifies multi-step logic, contradiction detection, and inference chains.

6.3 Model Tier Mapping

The Small Tier covers complexity scores 1 through 4. Models include Llama 3 8B, Claude Haiku, and Gemini Flash at approximately 0.0001 to 0.001 dollars per thousand tokens. Typical use cases are extraction, formatting, parsing, and summarisation.

The Large Tier covers complexity scores 5 through 7. Models include GPT-4o and Claude Sonnet 3.5 at approximately 0.005 to 0.015 dollars per thousand tokens. Typical use cases are analysis, multi-document synthesis, and code generation.

The Reasoning Tier covers complexity scores 8 through 10. Models include o3-mini and Claude Opus 4 at approximately 0.015 to 0.060 dollars per thousand tokens. Typical use cases are deep logic, contradiction detection, and complex planning.

7. DETAILED FUNCTIONAL REQUIREMENTS

FR-01: Complexity Classifier

The system shall score every incoming prompt on a 1 to 10 scale before any LLM call is made. It shall use a lightweight ML classifier based on scikit-learn or a rule-based fallback with less than 5 milliseconds of inference time. It shall expose a scoring breakdown per dimension for display on the dashboard. It shall support manual override via explicit task metadata such as force_tier equals large.

FR-02: Dynamic Model Router

The system shall dispatch to the correct model tier based on the complexity score. It shall use LiteLLM as the unified model proxy for all provider calls. It shall support adding or removing models from tiers via a configuration file with no code changes required. It shall implement a budget cap that halts dispatch when cumulative run cost exceeds the configured threshold.

FR-03: Semantic Cache

The system shall compute an embedding for every incoming prompt and query the vector database. It shall return a cached response immediately when cosine similarity exceeds 0.95. It shall log cache hits and misses in Langfuse with a zero dollar cost entry for all cache hits.

FR-04: Auto-Fallback and Self-Healing

The system shall validate model output using DSPy assertions covering schema compliance, length, and coherence score. It shall automatically escalate to the next tier when an assertion fails, moving from Small to Large to Reasoning. It shall log all fallback events including the original tier, the failure reason, and the escalated tier. It shall not retry the same tier more than once per node.

FR-05: Observability and Telemetry

The system shall send a Langfuse trace for every pipeline node containing the node ID, model used, complexity score, tier selected, routing reason, latency in milliseconds, token count, cost in USD, and whether a fallback was triggered. It shall maintain a real-time dashboard refreshing every 5 seconds. It shall compute and display the total cost for RouteIQ versus the all-large baseline, the savings percentage, and the quality delta.

FR-06: Demo Pipeline

The system shall implement a 5-node LangGraph pipeline covering Query Parsing, Web Search Summarisation, Evidence Analysis, Contradiction Detection, and Final Formatting. It shall demonstrate all three model tiers being used across the five nodes. It shall display per-node model selection and cost in the live dashboard.

11. FEASIBILITY AND RISK ANALYSIS

11.1 Technical Feasibility

All required frameworks including LangGraph, RouteLLM, LiteLLM, and Langfuse are mature, production-tested, and open source. All LLM tiers are accessible via existing public APIs with no infrastructure setup required. Complexity classification is solvable with rule-based logic for the hackathon scope, with a fine-tuned ML classifier as a post-hackathon enhancement. The system can be integrated into existing AI applications without major infrastructure changes and supports real-time routing decisions with minimal latency overhead.

11.2 Risk Register

Risk 1: The classifier mis-routes a high-complexity prompt to the Small tier, causing quality degradation. Likelihood is medium, severity is high. Mitigation is mandatory DSPy assertion fallback.

Risk 2: API rate limits are hit during the demo, causing pipeline stalls. Likelihood is medium, severity is medium. Mitigation is using a local Ollama Small tier as a backup.

Risk 3: Langfuse SaaS experiences downtime during the demo, eliminating observability. Likelihood is low, severity is medium. Mitigation is self-hosting Langfuse locally as a backup.

Risk 4: Semantic cache returns a false positive, serving a wrong cached answer. Likelihood is low, severity is high. Mitigation is setting the similarity threshold at 0.95 or higher and logging all cache hits for review.

14. GLOSSARY

LLM: Large Language Model, referring to AI models such as GPT-4o, Claude, and Gemini.

Routing Layer: Middleware that intercepts prompts and selects the appropriate LLM before any API call is made.

Complexity Score: A numerical rating from 1 to 10 measuring a task's difficulty across five weighted dimensions.

Model Tier: A group of LLMs with similar cost and capability profiles, organised as Small, Large, or Reasoning.

Semantic Cache: A vector database storing past prompt embeddings to return cached answers for similar future queries.

Auto-Fallback: Automatic escalation to a stronger model when the current model's output fails quality assertions.

Langfuse: An open-source LLM observability platform for tracing, cost tracking, and performance monitoring.

LiteLLM: A unified Python proxy for calling over 100 LLM APIs through a single interface.

LangGraph: A framework for building stateful, multi-step LLM pipelines as directed graphs.

DSPy: A framework for programming LLMs with typed assertions instead of hand-written prompts.

BERTScore: An NLP metric measuring semantic similarity between generated output and a reference text.
